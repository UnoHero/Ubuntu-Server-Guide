How to setupp VM Server

By following the video, you will learn how to create and set up your own VM server.
To begin, you will have to go to the IKT-Fag website. Open your preferred search engine and look for the IP 10.12.99.69, which will take you to the IKT-Fag website. Once there, in the upper right corner, you'll find a menu. Click on your class's name, and then click on your own username. There will be some information for later.
Under your username, you will find a link to VMWare vCenter. Open it in a new tab and log in with your "username"@ikt-fag.no and your password.
Now that you are inside, you need to find your class's name and then your Elev-Pool. If you don't know yours, you can check on the IKT-FAG website.
Right-click on your Elev-Pool and select "New Virtual Machine." This is where you give the VM a name, then click "Next." Make sure the VM is in the correct Elev-Pool, and then click "Next." Select "ikt-fag," and click "Next." Ensure it is on the correct version, and then click "Next."
Here is where you choose the operating system (OS) for your VM. We will use Ubuntu (64-bit) for this tutorial.
Since this will be a WebServer VM, we don't need a lot of processing power or RAM. You can set the RAM to 8GB if you want. Open the "Hard Disk" menu and find "Disk Provisioning," then change it to "Thin Provision." Now, open the "New CD/DVD Drive" menu and change the Client Device to "Content Library ISO File," which will open a menu where you can choose "ubuntu-22.04.1-live-server-amd64." There, now we are ready to start the server.
When the VM has started, click "LAUNCH WEB CONSOLE." Select "Try and install Ubuntu Server," and choose the language and keyboard layout you prefer.
When you get to the Internet page, go to the top and select "Edit IPv4." Change from "automatic (DHCP)" to "Manual." Here, you will need to fill out the following details to get Internet access:
Subnet: You can find the subnet information on your IKT-Fag page. It will look like x.x.x.0/24; write it in the subnet area.
Address: This is the personal IP for the VM. The first three parts of the IP should match your subnet. The last part should be 69 for others to find your site.
Gateway: You can find your Gateway on the same page as the Subnet.
NameServer: Your NameServer should start with 10.10.1.30, which is the school's DNS. If you want, you can add more DNS servers like 1.1.1.1 or 8.8.8.8.
Save your settings and continue.
You should "Update to the new Installer."
Use the entire disk, and after that, fill out your name, desired username, and password.
Don't select Ubuntu Pro, if you want you can install ssh and log in with you github user. Restart the server when the installation is complete. Press Enter to remove the installation medium and log in when it's ready.
There you have it - an up and running Ubuntu VM!

How to setupp and install Ssh and Git

To install Ssh you have to run the comand "sudo apt install openssh-server".
To check if ssh works run the comand "sudo systemctl status ssh"

Open a new terminal on your computer. Write "ssh username@x.x.x.69".
The username is the same as your VM's.
After the @ symbol you have to place the IP address of the VM.
Press enter, user the Vm's password to loginn.

Now that you are inside the Vm you have to get the code you wil be displaying on your website.
To get this we wil be using GitHub. This means that you can write and make the code in a diferent place and download it to your server.
First we will need to install Git on our VM, write "sudo apt install git".
Now that you have git we only need to grab the code.
If you dont have a GitHub profile i recomend you make one. Or you can download mine.

Apache2

Befour we can get our code we have to install Apache2 witch wil run our website.
Run "sudo apt install apache2" to install Apache2.
Apache2 gets its code from the folder "/var/www/html". In there it looks for a folder named index.html.
After installing Apache2 if you open the browser of your choise and seartch for your servers IP-adress, you shud get the defoult Apache2 site.
