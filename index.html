How to setup VM Server

Setting up Your VM Server:

  • Follow the video tutorial to create and set up your VM server...

Accessing the IKT-Fag Website:

  • Go to the IKT-Fag website by searching for the IP address 10.12.99.69 in your preferred search engine.
  • Click on the menu in the upper right corner and select your class's name, then your own username for important information.

Accessing VMWare vCenter:

  • Find the link to VMWare vCenter under your username.
  • Open it in a new tab and log in using your "username"@ikt-fag.no and password.

Creating Your VM:

  • A Virtual Machine is like a computer within a computer. It's a pretend computer that runs on your real computer. This pretend computer can have its own operating system and software, just like a real one. People use VMs to do different tasks on the same computer without them getting in each other's way. It's like having multiple computers in one.
  • Locate your class's name and Elev-Pool within VMWare vCenter.
  • Right-click on your Elev-Pool and choose "New Virtual Machine."
  • Provide a name for your VM and proceed through the setup steps, ensuring it's in the right Elev-Pool and on the correct version.
  • Select Ubuntu (64-bit) as the operating system.
  • An operating system is like the boss of your computer. It tells your computer what to do and how to do it. It manages all the programs and files, and it helps you interact with your computer, like clicking on icons and opening apps. Without an operating system, your computer wouldn't know how to work.

Configuring VM Settings:

  • Adjust the VM's resources, such as RAM (Random Accses Memory) you can set it to 8GB.
  • Configure the "Hard Disk" settings to use "Thin Provision". Thin Provisioning is like a flexible storage plan for your computer. Instead of reserving a lot of space upfront, it only uses what you need. It's like saying you have a big backpack, but you only put stuff in it when you actually need it. This helps save space and makes your computer more efficient.
  • Set the "New CD/DVD Drive" to "Content Library ISO File" and choose "ubuntu-22.04.1-live-server-amd64."

Installing Ubuntu Server:

  • Launch the web console and select "Try and install Ubuntu Server."
  • Choose your preferred language and keyboard layout.
  • When prompted, edit the IPv4 settings, changing from "automatic (DHCP)" to "Manual."
  • In simple terms, DHCP is a system that assigns unique addresses to your devices on a network, making it easier for them to connect to the internet without you having to set up each device individually. It's like a party host arranging seats for guests.
  • Fill in the Subnet, Address, Gateway, and NameServer details as required.
  • Think of a subnet as a neighborhood in a city. It's a group of connected devices or computers that can easily talk to each other because they are in the same 'neighborhood.' It helps organize and manage devices on a network.
  • An address is like your home address but for a computer on a network. It's a unique set of numbers that tells other devices where to find or send data. It's like a computer's location on the internet.
  • A gateway is like the main entrance to a neighborhood. It's the connection point between your local network and the outside internet. It directs data in and out, acting as the 'door' for your network to communicate with the broader internet.
  • A NameServer is like a phone book for the internet. It translates easy-to-remember names (like website URLs) into IP addresses, which computers understand. It helps your computer find the right location on the internet when you type a web address.

Final Steps:

  • "Update to the new Installer."
  • Use the entire disk and complete the installation by entering your name, desired username, and password.
  • Do not select Ubuntu Pro, you can install SSH for GitHub if you want.
  • Restart the server after installation, press Enter to remove the installation medium, and log in when it's ready.

Result:

  • Congratulations! You now have an up and running Ubuntu VM.

How to setupp and install Ssh and Git

Installing SSH:

  • Secure Shell or SSH is a secure way to access and control your computer from a distance, like having a magic key for remote access that keeps your computer safe.
  • Run the following command to install SSH: "sudo apt install openssh-server"...

Command Explanation

  • Sudo is like a special command that gives you superpowers on your computer. It stands for 'Superuser Do' and lets you do important things, like making big changes or installing software, that regular users can't.
  • Apt is a tool that helps you find and install software on your computer, acting like a helpful robot that makes software management simple.

Checking SSH Status:

  • To verify if SSH is working correctly, execute the command: "sudo systemctl status ssh."

Logging into Your VM:

  • Open a new terminal on your computer...
  • Type the following command: "ssh username@x.x.x.69" into the terminal.
  • Replace "username" with your VM's username.
  • After the "@" symbol, enter your VM's IP address.
  • Press Enter, and use your VM's password to log in.

Getting Your Website Code:

  • Now that you're inside the VM, it's time to access the code you'll display on your website...
  • GitHub will be our tool of choice, allowing you to create and edit code in a separate location before downloading it to your server.
  • Start by installing Git on your VM. Execute the command: "sudo apt install git."
  • With Git installed, you're ready to fetch the code you need.
  • If you don't have a GitHub profile yet, I recommend creating one. Alternatively, you can download mine.
  • If you don't want to install GIT you can use the command "SCP". (Secure Copy Protocol) To transfer files from one computer to another.

Apache2

Setting Up Apache2 for Your Website:

  • Before we can access our code, we must install Apache2, the web server that will host our website...
  • Apache2 is software that manages web traffic and makes websites accessible on the internet, like a helpful guide for your computer to display web pages.
  • To install Apache2, run the command: "sudo apt install apache2."
  • Apache2 fetches its content from the directory "/var/www/html." Within this folder, it specifically looks for a folder named "index.html."
  • Once Apache2 is installed, you can open your preferred web browser and search for your server's IP address. You should then see the default Apache2 site.

Github

Hosting Your GitHub Repository on Your Website:...

  • Open the GitHub repository you intend to host on your website Find the SSH link and copy it.
  • You can use the HTTPS link, it wont require a key so it can be faster/easyer but if it's a site that is going to be changed and uppdated a lot ssh is better.

Setting Up Your Server:

  • In the server's terminal, navigate to /var/www/ but refrain from entering the html folder as it will be removed later.
  • Attempt to clone the repository by running "sudo git clone (the copied link)." This wil download the files from github.
  • If you encounter an error, it's likely due to a missing SSH key.

Generating an SSH Key:

  • To generate an SSH key, go to the root directory and then to ".ssh." Start by running "sudo -i" to log in as the root user.
  • In the "/root/.ssh" directory, use the command "ssh-keygen" and press Enter for each question.
  • Confirm if it worked by running "ls." If you see a file named "id_rsa.pub," you've done it correctly.
  • Print the key to the terminal by running "cat id_rsa.pub." Copy everything from "ssh-rsa" to "@test."

Adding the SSH Key to GitHub:

  • Open your GitHub settings, locate the SSH key section, and click "create a new key."
  • Paste the key into the large box and assign it a name. Save it, and then reopen the terminal.

Final Steps:

  • Type "exit" to exit from the root user. Ensure you're in the /var/www/ directory, not /var/www/html.
  • Try cloning the repository again using "sudo git clone (the GitHub SSH link you copied)." If successful, it will begin downloading, and you'll see it if you run "ls."
  • Apache2 searches for the "html" folder, so we must remove the existing one and rename the GitHub folder. Execute "sudo rm -r html" to delete the html folder and its contents.
  • To ensure Apache2 finds your site, rename the downloaded folder to "html" using "sudo mv /var/www/(Name of the folder) /var/www/html."
  • Confirm that your main HTML file is named "index.html," or Apache2 won't locate it.
  • Now, try visiting your website using the IP address, and your website should appear.
